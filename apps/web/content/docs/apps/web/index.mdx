## Overview

This app combines a standard Next.js frontend with a Fumadocs-powered documentation site. It consumes shared packages from the monorepo, renders the docs under `content/docs`, and includes helper scripts for materializing external documentation into committed markdown files.

## Development

```bash
pnpm --filter @md-oss/web dev
pnpm --filter @md-oss/web build
pnpm --filter @md-oss/web start
pnpm --filter @md-oss/web typecheck
pnpm --filter @md-oss/web lint
```

By default the development server runs on `http://localhost:3000`.

## Docs Pipeline

The web app uses Fumadocs with `content/docs` as the source directory.

- `pnpm --filter @md-oss/web docs:sync` materializes external docs into committed files under `content/docs`.
- `pnpm --filter @md-oss/web docs:invariants` checks docs layout assumptions and manifest integrity.
- `postinstall` runs `fumadocs-mdx` so the MDX source metadata is generated locally.

Production builds should rely on committed documentation content already present in the repository rather than syncing docs at runtime.

## Structure

- `src/app` contains the Next.js app routes.
- `src/components` contains reusable UI components.
- `src/layouts` contains shared page and docs layout primitives.
- `src/lib` contains app-level helpers and content source wiring.
- `content/docs` stores the documentation content rendered by Fumadocs.
- `scripts/materialize-external-docs.mjs` imports external markdown into the docs tree.
- `source.config.ts` defines the Fumadocs content configuration.
